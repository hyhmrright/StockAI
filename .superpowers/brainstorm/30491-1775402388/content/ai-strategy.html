最后确认：AI 模型对接策略

获取到股票数据和全网信息后，我们需要一个强大的 AI 大脑来进行“综合分析得出未来走势和购买建议”。在桌面客户端中，您希望如何提供这个 AI 大脑？

A

自带密钥模式 (BYOK - Bring Your Own Key) 【推荐】

软件自带一个设置界面，让用户填入自己的 OpenAI (ChatGPT)、Anthropic (Claude) 或 DeepSeek 的 API Key。这也是 Cherry Studio 的核心做法。

优势

  • 您作为开发者不需要承担任何 AI 推理费用
  • 用户可以使用目前世界上最强的模型来分析股票

劣势

  • 对小白用户有一定的门槛 (需要知道怎么获取 Key)
B

完全本地模型模式 (Ollama)

完全免费、断网可用、隐私极佳！软件直接调用用户电脑上安装的 Ollama (例如运行 Llama 3 或 Qwen 2)。

优势

  • 完全免费，无需注册任何账号，财务数据绝不上传云端

劣势

  • 极其吃电脑配置，普通轻薄本跑不动
  • 小参数模型的逻辑推理能力比不上 GPT-4o 或 Claude 3.5
C

混合模式 (云端 API + 本地模型)

在设置里，用户可以下拉选择：是用自己填的 OpenAI Key，还是用本地的 Ollama 服务。给用户最大的自由度。

优势

  • 极度灵活，照顾到不同需求的高级用户

劣势

  • 开发工作量稍微增加（需要对接不同格式的 API，不过有很多现成库可以解决）
