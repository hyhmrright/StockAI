# 设计文档：搜索建议实时行情显示

## 1. 背景与目标
为了提升用户的搜索体验，使用户在输入股票代码或名称时就能快速了解该股票的当前市场价格和走势，我们需要在搜索建议下拉列表中集成实时行情数据。

## 2. 核心设计
### 2.1 数据获取策略
- **搜索建议**：继续使用新浪财经 `suggest3` 接口。
- **批量行情**：
    - **A 股**：使用新浪简易行情接口 `https://hq.sinajs.cn/list=s_sh601012,s_sz000001`。
    - **美股**：使用新浪美股行情接口 `https://hq.sinajs.cn/list=gb_aapl,gb_tsla`。
- **并发处理**：在 Sidecar 中异步并行请求行情数据，并根据 `fullCode` 将行情合并至搜索建议对象中。

### 2.2 前端展示逻辑
- **布局**：搜索建议项采用两端对齐布局。左侧显示名称和代码/类型，右侧显示价格和涨跌幅。
- **配色**：
    - 涨幅 ≥ 0：使用 `text-emerald-500` (绿色/上涨)。
    - 跌幅 < 0：使用 `text-rose-500` (红色/下跌)。
- **健壮性**：对于部分无行情数据的项（如部分指数或停牌股），优雅隐藏行情区域。

## 3. 技术实现
- **文件变更**：
    - `shared/types.ts`: 扩展 `StockSearchResult` 类型。
    - `sidecar/search.ts`: 实现 `enrichWithQuotes` 逻辑。
    - `src/components/SearchHeader.tsx`: 更新渲染逻辑。

## 4. 验证方案
- **单元测试**: `sidecar/search.test.ts` 验证字段是否存在。
- **集成验证**: 通过 Sidecar CLI `--search` 命令验证数据正确性。
