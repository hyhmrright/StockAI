# Design Doc: Fundamental Fix for StockAI macOS Bundle (aarch64)

## Context
The user reported that the `StockAI_*_aarch64.dmg` generated by GitHub CI has issues (likely "damaged" or signing-related). This project uses Tauri 2.0 with a Bun-based sidecar.

## Root Cause Analysis
1.  **Bun Signature Conflict**: Bun 1.2+'s default ad-hoc signing (`bun build --compile`) creates a Mach-O structure that often conflicts with manual `codesign` used for Hardened Runtime/Notarization.
2.  **Attribute Pollution**: Files in the CI environment may have extended attributes (`xattr`) that cause Gatekeeper failures if not cleared before bundling.
3.  **Entitlements Inheritance**: The sidecar needs to be explicitly signed with the same entitlements as the main app to ensure it can run JIT/Network tasks within the sandbox.

## Proposed Changes

### 1. Sidecar Build Script (`sidecar/build-script.ts`)
- Set `BUN_NO_CODESIGN_MACHO_BINARY=1` during compilation.
- Ensure `xattr -cr` is called on the binary before `codesign`.
- Apply `codesign` with `--options runtime`, `--timestamp`, and `--entitlements`.

### 2. GitHub Release Workflow (`.github/workflows/release.yml`)
- Add a "Recursive Attribute Cleanup" step before `tauri-action`.
- Pass all required Apple signing secrets to the build step.
- Enhance the verification step to mount the DMG and perform `spctl` and `codesign -vvv --deep` checks.

### 3. Tauri Configuration (`src-tauri/tauri.conf.json`)
- Verify `bundle.macOS.hardenedRuntime` is `true`.
- Ensure `bundle.macOS.entitlements` points to the correct `Entitlements.plist`.

## Success Criteria
- CI produces a DMG that passes `spctl --assess --verbose` on macOS.
- The internal sidecar binary passes `codesign -vvv --deep --strict`.
- The app runs without "damaged" warnings on a fresh macOS installation (assuming notarization succeeds).

## Risk Assessment
- **Notarization Delays**: Apple's notarization servers can sometimes be slow, potentially timing out the CI job.
- **Entitlements Mismatch**: If the main app and sidecar have conflicting entitlements, the sidecar might be killed at runtime. We will use the same `Entitlements.plist` for both.
